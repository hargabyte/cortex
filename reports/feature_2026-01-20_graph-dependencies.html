Feature Overview

The Graph & Dependencies feature provides comprehensive analysis of code structure, function calls, and dependency relationships across the codebase. It enables developers and tech leads to understand code flow, identify bottlenecks, trace execution paths, and detect circular dependencies.

Key capabilities include:

  • Call Graph Analysis: Extract and visualize function/method call relationships
  • Dependency Tracing: Trace execution paths between functions with configurable hop limits
  • Graph Algorithms: Apply PageRank and Betweenness Centrality for importance scoring
  • Cycle Detection: Identify circular dependencies and problematic patterns
  • Multi-Language Support: Works with Go, TypeScript/JavaScript, Ruby, and more
  • Bottleneck Identification: Find critical nodes with high in-degree or betweenness scores

Graph Algorithms

Cortex implements sophisticated algorithms for analyzing code structure and identifying critical components.

PageRank Algorithm

Function: ComputePageRankWithInfo (pagerank.go:55-130)

Importance scoring via random walk simulation. Assigns scores based on the principle that important entities are called by other important entities. Used to identify high-value functions and modules that are frequently referenced across the codebase.

Use Case: Finding critical business logic functions that are depended upon by many other components.

Betweenness Centrality

Function: ComputeBetweenness (betweenness.go:25-106)

Bottleneck detection via shortest path analysis. Measures how often a function appears on the shortest path between other entities. High betweenness scores indicate functions that are critical "bridge" points in the code flow.

Use Case: Identifying bottlenecks and pinch points where changes have maximum impact. Essential for refactoring planning.

Dependency Tracing

Function: traceFlow (smart.go:490-707)

Bidirectional tracing of execution paths with configurable hop limits. Finds all call chains from one function to another, respecting depth constraints and cycle detection.

Use Case: Understanding call chains for debugging, impact analysis, and feature propagation paths.

Technical Recommendations

1. Leverage PageRank for Priority Selection

When planning refactoring or optimization work, prioritize entities with high PageRank scores computed by ComputePageRankWithInfo. These represent the most frequently depended-upon code.

2. Monitor CallGraphEntity Bottleneck

The CallGraphEntity type has an in-degree of 86, making it a critical bottleneck. Any changes to this type could have widespread impact. Ensure comprehensive test coverage before modifying.

3. Use Betweenness Centrality for Test Planning

Functions with high betweenness centrality deserve extra testing attention. Apply ComputeBetweenness to identify which functions should have the most comprehensive test coverage.

4. Implement Recursive Tracing Limits

The TraceCallers function is recursive. Always use configurable hop limits to prevent excessive traversal and maintain performance, especially in large codebases.

5. Regular Cycle Detection

Incorporate runGuideDeps into your CI/CD pipeline to detect circular dependencies early. Circular dependencies can cause initialization issues and testing complications.

6. Archive Pre-Graph Diagram Scans

Use runGuideModules to generate architectural diagrams. Archive these periodically to track how your architecture evolves over time.
