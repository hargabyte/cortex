How to Use Change Reports

Time-Travel Queries

CX uses Dolt's time-travel capability to compare the code graph at different points in time. This enables:

  • Comparing code structure across releases
  • Understanding what changed in a feature branch
  • Auditing architecture changes over time

Example Commands

Generate change reports with different ranges:

# Compare last 50 commits
cx report changes --since HEAD~50 --data

# Compare between tags
cx report changes --since v1.0 --until v2.0 --data

# Compare with a specific commit
cx report changes --since abc1234 --data
