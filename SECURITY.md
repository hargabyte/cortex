# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Cortex, please report it responsibly:

1. **Do NOT open a public issue** for security vulnerabilities
2. Email the maintainer directly at hargabyte@gmail.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## What to Expect

- Acknowledgment within 48 hours
- Regular updates on progress
- Credit in release notes (unless you prefer anonymity)

## Scope

Security issues we care about:
- Code execution vulnerabilities
- Path traversal issues
- Sensitive data exposure
- Dependency vulnerabilities with known exploits

Out of scope:
- Issues requiring physical access
- Social engineering
- Denial of service (cx is a local CLI tool)

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| Best effort        |

We recommend always using the latest release.
